Research collaboration request

Debugging scenarios in Behavioral Programming

We are collecting representative types of bugs in Behavioral Programming programs to support research on LLM-agent based debugging workflows for BP models.

The four examples below are candidate bug types we found so far. We are especially interested in additional types of BP bugs: different failure modes, modeling mistakes, specification gaps, or recurring patterns that should appear in a future dataset.

Review the scenarios

The task

Help us expand the bug-type catalog

What to send

Any distinct BP bug type or pattern in free form. Concrete code, traces, screenshots, diagrams, or short notes are welcome when they help explain the type.

Timing

We plan to keep this collection phase open for two weeks. A reminder will be sent after about a week and a half.

Goal

The goal is to identify a broader set of BP bug types. From these types, we will create a dataset with varied concrete bugs and use it in debugging experiments.

Methodology

Our methodology studies how agentic LLM workflows inspect BP event traces, reason about synchronization behavior, explain faults, and propose repairs.

Acknowledgments

Thank you for helping shape the bug-type catalog

We thank the BP community, researchers, lecturers, and practitioners who review these examples and contribute additional bug types. Concrete examples are useful, but the main value is identifying distinct kinds of BP faults. Your feedback will help make the resulting dataset more realistic and useful.
